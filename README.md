# 產險理賠 AI 自動化實作：Codex 模擬資料集

本儲存庫（Repository）包含了一系列模擬資料，旨在展示 **Codex (AI 代理人)** 如何賦能產險理賠專員，解決傳統人工核對中「耗時、易錯、資訊破碎」的痛點，實現智慧化的理賠審核自動化。

## 🎯 專案目標

透過 Codex 的強大理解與推論能力，學員將學習如何：
1. **跨文件比對**：自動對照「理賠作業準則」與「車廠估價單/收據」。
2. **自動核算**：辨識哪些項目可理賠、哪些須剔除，並精確計算最終理賠金額。
3. **自動排版**：將審核結果直接填入公司標準 HTML 公文格式。
4. **大規模資料處理**：學習如何將雜亂的原始資料整理成標準化結構，並在資料量增加 10 倍時保持高效率。

---

## 📂 目錄結構說明

### 1. 實作二_模擬資料_A (個人理賠審核實戰)
此資料夾對應課程中的 **實作二**，模擬典型的個人理賠案件審核流程。
- **用戶資料**：包含車廠估價單、交通事故報案單、機票、電子機票、收據與信用卡帳單等雜亂附件。
- **理賠作業準則手冊**：提供「車體損失保險」、「旅遊不便險」、「寵物綜合保險」三種險種的官方審核標準。
- **公司公文格式**：包含標準的報告模板（`report_layout.html`）。
- **核心挑戰**：讓 Codex 讀懂準則後，判斷估價單是否「灌水」，並產出最終 HTML 審核公文。

### 2. 實作三_模擬資料_B (進階理賠審核實戰)
此資料夾對應課程中的 **實作三**，資料量為實作二的 **10 倍**，測試 Codex 處理大規模資料的效能。
- **原始資料**：包含大量保戶的 PDF 圖片與 Markdown 格式的基本資料。
- **任務目標**：學員需引導 Codex 將這些散亂的檔案進行整理與分類。

### 3. 實作三_模擬資料_B_目標結構
這是 **實作三** 的預期產出範本。
- 學員需練習如何透過 Codex 把「實作三_模擬資料_B」整理成此目標架構（按保戶姓名/案件編號分類）。
- 在整理完成的架構中，接續進行自動化理賠審核與公文產出。

## 🛠 實作指引

### 實作二：個人理賠自動化
1. **輸入**：將 `實作二_模擬資料_A` 內的附件、準則手冊與模板餵給 Codex。
2. **撰寫 Prompt**：
   - 指令範例：「請閱讀理賠準則，對照車廠估價單，剔除不屬於本次事故的維修項目，計算總金額，並以公司公文格式產出報告。」
3. **驗證**：確認 AI 計算的理賠金額是否準確，且 HTML 排版無跑版。

### 實作三：進階資料整理與擴展
1. **資料整理**：操作 Codex 將 `實作三_模擬資料_B` 的雜亂檔案整理至 `實作三_模擬資料_B_目標結構`。
2. **壓力測試**：在 10 倍資料量下，觀察 Codex 的處理速度與準確度。
3. **效率對比**：討論與人工審核相比，處理時間與人力成本的節省程度。

## 📊 預期效益
- **減少人工誤差**：避免人工翻閱條款時的疏漏。
- **極大化處理能量**：資料量增加時，僅需調整 Prompt 或自動化流程，無需成倍增加人力。
- **標準化產出**：確保每一份理賠公文都符合公司規定的排版與敘述邏輯。
